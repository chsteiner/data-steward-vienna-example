OpenRefine Data Cleaning Guide

A comprehensive step-by-step guide to fixing all data quality issues in the US Presidents dataset using OpenRefine's powerful transformation tools.

45 min read 10 Phases 47 Records

00Dataset Overview

This guide addresses a dataset containing information about US Presidents that has been intentionally corrupted with various data quality issues – perfect for learning OpenRefine's data cleaning capabilities.

01Importing the Data

Step 1.1: Create a New Project

1

Open OpenRefine

Navigate to http://localhost:3333 in your browser. Click Create Project → Choose Files and select your CSV file.

Step 1.2: Configure Import Settings

Critical Settings

Incorrect import settings can introduce additional data corruption. Verify these settings before proceeding.

Setting Value Reason
Character encoding UTF-8 Prevents additional encoding corruption
Columns separated by comma Standard CSV delimiter
Parse column headers ✓ Checked First row contains headers
Trim whitespace ✓ Checked Removes obvious whitespace issues

02Whitespace Cleanup

Whitespace issues affect multiple columns and can cause matching and grouping failures. We'll clean them systematically.

Step 2.1: Trim All Columns

For each text column (president, prior, party, vice):

→

Click the column dropdown → Edit cells → Common transforms → Trim leading and trailing whitespace

This handles cases like:

  • Aaron Burr → Aaron Burr
  • Ulysses S. Grant → Ulysses S. Grant
  • Rutherford B. Hayes → Rutherford B. Hayes

Step 2.2: Collapse Internal Whitespace

For columns with multiple internal spaces:

→

Click the column dropdown → Edit cells → Common transforms → Collapse consecutive whitespace

This fixes:

  • George Clinton → George Clinton
  • John C. Calhoun → John C. Calhoun

Step 2.3: Verify Cleanup

Create a custom text facet to check for remaining issues:

value.startsWith(" ") or value.endsWith(" ")GREL

If any results show true, there's still whitespace to clean.

03Character Encoding Fixes

The dataset has corrupted en-dash characters (â€" instead of –). This is a classic UTF-8 encoding issue.

Step 3.1: Fix the Encoding Issue

Apply this transformation to the prior column:

1

Open Transform Dialog

Click prior dropdown → Edit cells → Transform

2

Enter GREL Expression

value.replace("â€"", "–")GREL

Alternative: Use Standard Hyphen

If you prefer ASCII-only output:

value.replace("â€"", "-")GREL
Tip

Check other columns for the same encoding issue using a text facet. Look for any entries containing â€" or similar corrupted characters.

04Capitalization Normalization

The president column has inconsistent capitalization that will cause grouping failures:

  • john adams (all lowercase)
  • JAMES MONROE (all uppercase)
  • john tyler (all lowercase)

Step 4.1: Apply Title Case

1

Transform to Title Case

Click president dropdown → Edit cells → Transform

value.toTitlecase()GREL

Result:

  • john adams → John Adams
  • JAMES MONROE → James Monroe
  • john tyler → John Tyler

Step 4.2: Verify Results

Create a text facet on the president column and review all values to ensure proper capitalization.

05Clustering for Spelling Errors

OpenRefine's clustering feature is the most powerful tool for finding and fixing spelling variations.

Step 5.1: Open Clustering

1

Click party dropdown → Facet → Text facet

In the facet panel, click Cluster

Step 5.2: Clustering Methods

Try these methods in sequence for best results:

Method 1: Key Collision - Fingerprint

Best for finding variations with whitespace/case differences.

Method 2: Key Collision - N-gram Fingerprint

Good for finding variants with minor spelling differences. Set N-gram size to 2.

Method 3: Nearest Neighbor - Levenshtein

Best for catching typos. Set radius to 2 to catch 1-2 character differences.

Expected Clusters

Cluster Values Found Merge To
Democratic Democratic, Demcratic, Democatic, Demorcatic Democratic
Republican Republican, Replican, Republn, Repblican, Republicans Republican
Whig Whig, Wig Whig
Democratic-Republican Multiple spacing variants Democratic-Republican

Step 5.3: Merge Clusters

For each cluster:

  1. Check the box to select the cluster
  2. Choose the correct value in the New Cell Value field
  3. Click Merge Selected & Re-Cluster

06Party Name Standardization

After clustering, you may still have formatting variations for "Democratic-Republican":

  • Democratic - Republican (spaces around hyphen)
  • Democratic- Republican (space after only)
  • Democratic -Republican (multiple spaces)

Step 6.1: Standardize with GREL

if(value.contains("Democratic") and value.contains("Republican"),
   "Democratic-Republican",
   value)GREL

Alternative: Regex Approach

value.replace(/Democratic\s*-\s*Republican/, "Democratic-Republican")GREL

Step 6.2: Verify Party Column

Create a text facet and confirm you have only valid party names:

  • Nonpartisan
  • Federalist
  • Democratic-Republican
  • Democratic
  • Whig
  • Republican

07Duplicate Detection & Removal

Step 7.1: Identify Duplicates

1

Click S.No. dropdown → Facet → Customized facets → Duplicates facet

Click true to show only duplicate rows.

You should find:

  • S.No. 28 appears twice (Woodrow Wilson)
  • S.No. 44 appears twice (Barack Obama / Barak Obama)

Step 7.2: Star Rows to Keep

2

In the duplicates view, star (⭐) the row you want to KEEP:

  • For Woodrow Wilson: star either row (both are equivalent after party fix)
  • For Barack Obama: star the row with correct spelling "Barack Obama"

Step 7.3: Remove Unmarked Duplicates

3

Click All dropdown → Facet → Facet by star

Click false to show unstarred rows.

Verify these are the rows to delete, then:

All → Edit rows → Remove all matching rows

Clean Up

After removing duplicates, close all facets and remove stars: All → Edit rows → Unstar rows

08Date Format Standardization

The dataset uses two incompatible date formats:

Format Example Rows
DD.MM.YYYY 14.07.2021 2–45
German Long Mittwoch, 14. Juli 2021 46–48

Step 8.1: Fix Numeric Dates

Convert DD.MM.YYYY to ISO format YYYY-MM-DD:

if(value.match(/\d{2}\.\d{2}\.\d{4}/),
   value.split(".")[2] + "-" + value.split(".")[1] + "-" + value.split(".")[0],
   value)GREL

Step 8.2: Fix German Dates

For the German format dates, apply targeted fixes:

if(value.contains("Juli"), "2021-07-14", value)GREL
if(value.contains("Februar"), "2020-02-01", value)GREL

Step 8.3: Convert to Date Type

After standardizing formats:

Column dropdown → Edit cells → Common transforms → To date

09Number Format Fixes

The salary column uses European number formatting:

  • Current: $5.000,00 (period = thousands, comma = decimal)
  • Target: 5000.00 (standard numeric)

Step 9.1: Convert to Numeric

value.replace("$", "").replace(".", "").replace(",", ".")GREL

Result: $5.000,00 → 5000.00

Step 9.2: Convert to Number Type

Column dropdown → Edit cells → Common transforms → To number

Step 9.3: Reformat with Currency (Optional)

If you want US format with dollar sign:

"$" + value.toString().replace(/(\d)(?=(\d{3})+\.)/g, "$1,")GREL

10Structural Issues

Issue 1: John Tyler's Party Entry

Row 11 has embedded date information in the party field:

"Whig April 4, 1841 – September 13, 1841"

Fix with GREL:

if(value.startsWith("Whig "), "Whig", value)GREL

Issue 2: Fix S.No. Sequence

After removing duplicates, renumber the sequence:

row.index + 1GREL

Issue 3: Standardize "Office vacant"

Optionally transform to a consistent code:

if(value == "Office vacant", "VACANT", value)GREL

12GREL Quick Reference

Common Functions

Function Description Example
value.trim() Remove leading/trailing whitespace " text " → "text"
value.replace(a, b) Replace all occurrences value.replace("old", "new")
value.toTitlecase() Capitalize first letters "john adams" → "John Adams"
value.toUppercase() All uppercase "text" → "TEXT"
value.toLowercase() All lowercase "TEXT" → "text"
value.split(sep) Split into array "a,b,c".split(",") → ["a","b","c"]
value.contains(s) Check if contains Returns true or false
value.startsWith(s) Check prefix Returns true or false
value.length() String length "hello".length() → 5

Regular Expressions

// Replace with regex
value.replace(/\s+/, " ")  // Multiple spaces → single space

// Match pattern
value.match(/\d{4}/)  // Match 4 digits

// Test if matches
isNotNull(value.match(/pattern/))GREL

Conditional Logic

// Simple if
if(condition, valueIfTrue, valueIfFalse)

// Nested if
if(condition1, value1, 
   if(condition2, value2, value3))

// With and/or
if(condition1 and condition2, valueIfTrue, valueIfFalse)
if(condition1 or condition2, valueIfTrue, valueIfFalse)GREL

Type Conversions

value.toString()     // To string
value.toNumber()     // To number
value.toDate()       // To dateGREL
